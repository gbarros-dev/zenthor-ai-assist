# Zenthor AI Assist

Personal AI assistant for notes, reminders, and work management. Built on NanoClaw.

## Rules

- **NEVER modify `apps/backend/convex/_generated/`** — This folder is auto-generated by `convex dev`. Do not edit, create, or delete any files in it.

## Product Goal

A personal AI assistant accessible via WhatsApp (and optionally Telegram/Discord) focused on:
1. **Notes & Documentation** — Capture, organize, search, and retrieve notes (SQLite FTS5 + markdown backup)
2. **Reminders via Todoist** — Create, manage, and track tasks/reminders through the Todoist API
3. **Work Management via Linear** — Manage issues, track priorities through the Linear API (optional)

See [docs/PLAN.md](docs/PLAN.md) for the full implementation plan and phasing.

## Architecture

Bun workspaces + Turborepo monorepo with three apps and shared packages:

```
apps/ai-agent/    — WhatsApp AI agent (Node.js, Claude Agent SDK in containers)
apps/web/         — Next.js web UI (scaffold)
apps/backend/     — Convex backend (scaffold)
packages/config/  — Shared TypeScript config
packages/env/     — Zod environment validators (stub)
packages/observability/ — Logging (stub)
```

### AI Agent Flow
```
WhatsApp → SQLite → Poll loop (2s) → Trigger match → GroupQueue (max 5 concurrent)
→ Docker container → Claude Agent SDK (query()) → IPC → WhatsApp response
```

The agent inside the container has access to MCP tools (send_message, schedule_task, notes, todoist, linear) and the full Claude Code toolset (Bash, Read, Write, Edit, etc.) with permissions bypassed — the container IS the security boundary.

## Adding New MCP Tools

All product features follow one of two patterns:

**Pattern A — IPC-based (for local data, e.g. notes):**
1. Add tool definition in `apps/ai-agent/container/agent-runner/src/ipc-mcp-stdio.ts`
2. Tool writes JSON to `/workspace/ipc/{feature}/` directory
3. Add IPC handler in `apps/ai-agent/src/ipc.ts` to process the JSON files
4. Add database queries in `apps/ai-agent/src/db.ts`

**Pattern B — Direct API (for external services, e.g. todoist, linear):**
1. Create API client in `apps/ai-agent/container/agent-runner/src/{service}.ts`
2. Register tools in `apps/ai-agent/container/agent-runner/src/ipc-mcp-stdio.ts`
3. Pass API token via `apps/ai-agent/src/container-runner.ts` env passthrough

## Development

Run commands directly — don't tell the user to run them.

```bash
# Monorepo (from root)
bun install           # Install all workspace deps
turbo run build       # Build all workspaces
turbo run typecheck   # Type-check all workspaces
bun run test          # Run all tests

# AI Agent (from root or apps/ai-agent/)
cd apps/ai-agent
bun run dev           # Run with hot reload
bun run build         # Compile TypeScript
./container/build.sh  # Rebuild agent container
```

Service management:
```bash
# macOS (launchd)
launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist
launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist
launchctl kickstart -k gui/$(id -u)/com.nanoclaw  # restart

# Linux (systemd)
systemctl --user start nanoclaw
systemctl --user stop nanoclaw
systemctl --user restart nanoclaw
```

## Container Build Cache

The container buildkit caches the build context aggressively. `--no-cache` alone does NOT invalidate COPY steps — the builder's volume retains stale files. To force a truly clean rebuild, prune the builder then re-run `./container/build.sh`.
